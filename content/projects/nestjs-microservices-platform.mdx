## Overview

Designed and developed a production-grade NestJS microservices platform serving CRM, payment processing, and six additional business-critical modules. The platform supports a 14-engineer team delivering across three concurrent projects.

## Architecture

- **Event-driven design** using Azure EventHub and an internal EventBus across 10+ services
- **Dual-database strategy** with PostgreSQL for relational data and MongoDB for flexible document storage
- **Service boundaries** aligned with domain-driven design principles for independent deployability

## Key Results

- Eliminated ~30% of synchronous bottlenecks through async event processing
- Sub-200ms query performance on 500K+ records via strategic indexing
- 50% hands-on development contribution alongside technical leadership

## Challenges Solved

Coordinating schema evolution across services while maintaining backward compatibility, and establishing consistent observability patterns with Datadog APM across all microservices.
